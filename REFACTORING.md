# AdminStats Bot - Архитектурный рефакторинг

## Изменения от 27.04.2026

### Разделение модулей

Старый монолитный `cache.py` (204 строки) разделен на специализированные модули:

**utils/cache_manager.py** - Управление кэшем данных
- Загрузка и обновление данных из Google Sheets
- Поиск сотрудников в кэше
- Фоновое обновление с настраиваемым интервалом

**utils/rate_limiter.py** - Защита от спама
- Rate limiting на основе скользящего окна
- Потокобезопасная реализация

**utils/request_logger.py** - Логирование запросов
- История запросов пользователей (200 последних)
- Детекция подозрительной активности
- Алерты администратору

**utils/admin_notifier.py** - Уведомления админа
- Централизованная отправка сообщений администратору
- Обработка ошибок при отправке

**utils/card_constants.py** - Константы для генератора карточек
- Цветовая палитра
- Размеры и отступы
- Пороги для цветовой индикации
- Размеры шрифтов

### Преимущества новой архитектуры

✅ **Разделение ответственности** - каждый модуль решает одну задачу
✅ **Легче тестировать** - можно тестировать модули независимо
✅ **Проще поддерживать** - изменения в одном модуле не затрагивают другие
✅ **Переиспользование** - модули можно использовать в других проектах
✅ **Настраиваемость** - константы вынесены в отдельный файл

### Обратная совместимость

Все публичные API сохранены, изменились только импорты:

```python
# Было
from cache import find_employee_in_cache, check_rate_limit

# Стало
from utils.cache_manager import find_employee_in_cache
from utils.rate_limiter import check_rate_limit
```
